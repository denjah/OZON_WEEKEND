# Правила работы с проектом Ozon Manager

## Ozon API и данные по собственным товарам
- **Ozon API-Ключ:** API-ключ Ozon успешно получен, сохранен в конфигурации проекта и проверен.
- **Обновление данных:** Обновление данных по собственным товарам Weekend Billiard теперь выполняется исключительно через официальный Ozon API.
- **Парсинг/Скрапинг:** Сбор данных методом парсинга и скрапинга применяется только для анализа конкурентов. Для обновления данных собственных товаров скрапинг не используется.
- **Реализация интеграции:** Собственные товары подгружаются динамически через эндпоинт `/api/ozon/products` (объединяющий запросы к Analytics API и Product Info API Ozon). Хук `useOzonAnalytics` переведён с мок-данных Weekend на реальные данные из этого API-роута.

## SKILLS USE
В проекте Ozon Manager мы применяем кастомные скиллы (директория `/skills`) для сборки, аналитики и дизайна портала. Карта применения скиллов:

### 1. Сбор данных и конкурентная разведка (Scraping & Intelligence)
- **apify-ecommerce**, **apify-competitor-intelligence**, **apify-ultimate-scraper**, **web-scraper**
  - **Текущие задачи:** Сбор информации о ценах, отзывах и позиционировании конкурентов. (Помни: парсим только чужое, свое тянем по API Ozon).
  - **В будущем:** Настройка continuous-мониторинга цен, чтобы ловить демпинг конкурентов в реальном времени.

### 2. Аналитика и стратегия (Analysis & Strategy)
- **business-analyst**, **competitive-landscape**
  - **Текущие задачи:** Анализ рыночной ситуации, выстраивание структуры KPI-показателей.
  - **В будущем:** Моделирование ценообразования (dynamic pricing) и поиск неочевидных инсайтов в данных продаж.

### 3. Инженерия данных (Data Engineering)
- **data-engineer**
  - **Текущие задачи:** Проектирование нормализованных структур для хранения данных из Ozon API.
  - **В будущем:** Создание масштабируемых ETL-пайплайнов, интеграция Ozon-данных с внутренней ERP/1C и формирование единого Data Warehouse.

### 4. Дизайн и визуализация дашбордов (UI/UX)
- **kpi-dashboard-design**, **industrial-brutalist-ui**, **claude-d3js-skill**, **design-spells**
  - **Текущие задачи:** Построение брутального, информативного UI без лишнего мусора, с микро-анимациями (design spells).
  - **В будущем:** Внедрение сложных интерактивных визуализаций на D3.js (когортный анализ, графы связей товаров, тепловые карты заказов).

### 5. Отчетность и автоматизация (Reporting & Alerts)
- **pptx**, **python-pptx-generator**, **telegram-automation**
  - **Текущие задачи:** Настройка Telegram-ботов для алертов (например, упали остатки, пропал товар, прилетел плохой отзыв).
  - **В будущем:** Автогенерация еженедельных презентаций (PPTX) для руководства (цифры, графики, итоги) прямо из админки.

### 6. Декомпозиция и архитектура (Architecture)
- **blueprint**
  - **Текущие/В будущем:** Разбивка любых новых жирных фич на изолированные промпты-инструкции для ИИ-агентов, чтобы кодить параллельно и без конфликтов.
